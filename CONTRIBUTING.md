# Contributing

Thanks for taking a look at FinMAS.

## Maintainer Model

This repository is public, but it is currently maintained by a single primary
maintainer.

That means:

- issues are welcome
- pull requests are welcome
- review timing may vary
- roadmap and merge decisions remain maintainer-driven

Public visibility does not imply shared maintainer access.

## Best Way To Contribute

For bugs, missing docs, and feature requests, please open a GitHub issue first.

For larger changes:

- open an issue before investing in a major refactor or new subsystem
- keep pull requests scoped and easy to review
- include tests when behavior changes
- update docs when setup, behavior, or public expectations change

## Issues Vs Inline TODOs

Use GitHub issues for:

- bugs
- roadmap items
- user-facing limitations
- cross-file cleanup or design work

Use inline `TODO` comments only for small, code-local implementation notes that are
best understood directly next to the code.

## Public Repo Expectations

This workspace is still experimental. Some components are more mature than others, and
some flows still depend on optional local or cloud setup.

If you report a bug, it helps to include:

- the config or command you ran
- the expected behavior
- the actual behavior
- relevant logs or stack traces
