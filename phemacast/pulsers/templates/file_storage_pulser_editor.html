Operating Notes

A few sharp edges worth keeping visible while you tune the backend.
Filesystem mode Best for local development, quick testing, and single-host deployments. The pulser stores object bytes plus internal metadata documents under the configured root.
S3 mode Best for shared or durable storage. The pulser does not create real AWS buckets per logical bucket; it uses one backing bucket and enforces access in the agent layer.
Credential values For credential-like fields you can enter plain text, env:VAR_NAME, or a JSON object such as {"env":"AWS_ACCESS_KEY_ID"}.
